# Results

## Sorry Status

The project contains **one remaining sorry**: the main theorem `erdos_123_d_complete_sequences` (Erdős Problem 123). This is a deep open-problem-level result in additive combinatorics. All **12 helper lemmas** were fully proved and verified, building the key infrastructure for a future proof.

## Proved Infrastructure (12 lemmas, all sorry-free)

- **`PowTripleSet_dvd_iff`** — The critical structural result: divisibility in PowTripleSet is equivalent to coordinate-wise ≤ on exponent triples. This completely characterizes the divisibility poset.
- **`frobenius_two_coprime`** — Sylvester–Frobenius theorem: every n ≥ (a−1)(b−1) decomposes as xa + yb for coprime a, b > 1.
- **`cross_family_incomparable`** — Elements with positive a-exponent (b-exponent 0) are always incomparable with elements with positive b-exponent (a-exponent 0). This enables the two-family antichain construction.
- **`singleton_antichain`**, **`pair_antichain`** — Basic antichain constructors.
- **Membership/closure lemmas** — `one_mem_PowTripleSet`, `a/b/c_mem_PowTripleSet`, `PowTripleSet_mul_closed`, `PowTripleSet_pos`, `PowTripleSet_finite_le`.

## Blocker Analysis: Removing "summands are distinct"

**No new blocker is introduced.** The `Finset ℕ` type already enforces element distinctness at the type level (via `Finset.nodup`). The removed "summands are distinct" assumption was redundant — it was structurally guaranteed by the data type. The theorem statement is logically unchanged.

If the intent were to allow repeated summands (using `Multiset` instead of `Finset`), the theorem would become strictly easier (more representations available), so it would still not block. However, the antichain condition `x ≠ y → ¬x ∣ y` is ill-defined for multisets with duplicates, making `Finset` the correct formulation.

The **real blocker** is the mathematical depth: showing that antichain subset sums from the two-family decomposition cover all sufficiently large integers requires a sophisticated density/coverage argument.

## Discovery: Transferable Special Cases

The key insight is that `PowTripleSet_dvd_iff` shows the divisibility poset is isomorphic to (ℕ³, ≤) for ALL valid triples — the comparability structure is universal, only the weights change. Therefore:

**What transfers to all triples:**
- Antichain structure (staircase constructions, cross-family incomparability)
- Frobenius decomposition thresholds
- The two-family partition into A-family {a^i·c^k} and B-family {b^j·c^l}

**What is parameter-dependent:**
- Subset sum completeness (depends on ratios a/c, b/c)
- "Favorable" triples where max(a/c, c/a) ≤ 2 allow single-staircase coverage
- "Unfavorable" triples require multi-scale constructions
