# Erdős Problem 123: Reformulation Analysis

## Problem Statement

For pairwise coprime integers `a, b, c > 1`, is the set `{a^i · b^j · c^k : i,j,k ∈ ℕ}` *d-complete*? That is, can every sufficiently large integer be written as a sum of distinct elements from this set forming a **divisibility antichain** (no element divides another)?

## Is This Reformulation Easier or Harder?

**The reformulation is equivalent** to the standard Erdős Problem 123 statement. However, the antichain-sum-from-semigroup framing reveals structural insights that make the proof *architecturally cleaner*, even though the core difficulty is unchanged:

1. **Clearer**: The exponent-lattice viewpoint (Theorem `dvd_iff_exponents_le`) reduces divisibility to componentwise ≤ on ℕ³, separating the number theory from the combinatorics.

2. **Modular**: The scaling lemma (`represent_div_a`) cleanly handles the case when a generator divides `n`, reducing to a smaller problem.

3. **Reveals the obstruction**: The reformulation makes transparent *why* three generators are needed (see below).

## Key Discovery: Two Generators Can Fail

**The two-generator case `{a^i · b^j}` is NOT always d-complete.**

*Example*: `{8^i · 3^j}`. An antichain in ℕ² uses at most one element from each "row" (since `3^j₁ | 3^j₂` for `j₁ < j₂`). So any antichain sum is congruent to `0`, `1`, or `3` mod `8`. Integers `≡ 2 mod 8` are **never** representable.

This happens because the multiplicative subgroup `⟨3⟩` in `(ℤ/8ℤ)*` has order 2, generating only `{1, 3}` — a proper subgroup.

## Why Three Generators Suffice

With three generators, "row 0" (the `i=0` hyperplane) contains `{b^j · c^k}`, and **2D antichains** in the `(j,k)`-plane can have multiple elements. For example:

- `27 = 3³·5⁰` and `15 = 3¹·5¹` are incomparable (exponents `(3,0)` vs `(1,1)`)
- Their sum `42 ≡ 2 mod 8` recovers the missing residue class

More generally, for pairwise coprime `b,c > 1`:
- **Singletons** `{b^j · c^k}` cover the multiplicative subgroup `H = ⟨b,c⟩` of `(ℤ/aℤ)*`
- **Pairs** of incomparable elements cover `H + H` (sumset)
- **Triples** cover `H + H + H`, etc.

By the Cauchy-Davenport theorem (for prime `a`) or Kneser's theorem (for composite `a`), iterating sumsets eventually covers all of `ℤ/aℤ`. Since antichains of arbitrary size exist on anti-diagonals `{b^j · c^(N-j) : 0 ≤ j ≤ N}`, we can always find enough incomparable elements.

## Which Special Cases Transfer?

### Cases that transfer directly:
1. **`(2, 3, 5)` and permutations**: The prototypical case. Powers of 2 and 3 generate `(ℤ/5ℤ)*` entirely, so singleton coverage suffices.

2. **Any `(p, q, r)` with distinct primes**: If `q` is a primitive root mod `p`, then `⟨q⟩ = (ℤ/pℤ)*` and singletons cover all non-zero residues mod `p`. The scaling reduction (`represent_div_a`) handles multiples of `p`.

3. **Cases where `⟨b,c⟩ = (ℤ/aℤ)*`**: When the subgroup generated by `b` and `c` in `(ℤ/aℤ)*` is the full group, each residue class mod `a` has a singleton representative.

### Cases requiring sumsets (the novel insight):
4. **`(8, 3, 5)` and similar**: When `⟨b,c⟩ ⊊ (ℤ/aℤ)*`, we need *sums* of 2D antichain elements. The products `3^j · 5^k mod 8` give only `{1, 3, 5, 7}`, but pairwise sums recover `{0, 2, 4, 6}`.

5. **`(4, 9, 25)` and "perfect square" triples**: Even sparser sets, but the same mechanism works: 2D antichains provide enough combinatorial flexibility.

## Proof Architecture (Implemented)

### Fully proved lemmas:
- `dvd_iff_exponents_le`: Divisibility ↔ componentwise ≤ on exponents
- `antichain_iff_exponents_incomparable`: Antichain ↔ exponent incomparability
- `IsDComplete_mono`: Monotonicity of d-completeness
- `antichain_scale_a`: Scaling preserves antichains
- `scale_mem_PowTripleSet[_b/_c]`: Scaling preserves membership
- `represent_div_a`: Reduction when a generator divides n

### Remaining gaps:
- **`residue_coverage`**: For every residue `r mod a`, find an antichain from `{b^j · c^k}` whose sum hits `r`. This requires Cauchy-Davenport/Kneser-type arguments from additive combinatorics.
- **`erdos_123_d_complete_sequences`**: The main theorem, which follows from `residue_coverage` plus the scaling reduction.

## Proof Strategy for the Remaining Gaps

The main theorem follows from strong induction on `n`:
1. If `a | n`: use `represent_div_a` with IH on `n/a`
2. If `b | n` or `c | n`: symmetric scaling arguments
3. If `gcd(n, abc) = 1`: use `residue_coverage` to find antichain `s₀` from row 0 with `sum(s₀) ≡ n mod a`, then `a | (n - sum(s₀))`, reducing to case 1

For `residue_coverage`, the proof requires:
1. Show `H = ⟨b,c⟩` is a subgroup of `(ℤ/aℤ)*`
2. Show iterated sumsets `kH` cover all of `ℤ/aℤ` for `k` large enough
3. Construct explicit antichains (from anti-diagonals) achieving each residue
