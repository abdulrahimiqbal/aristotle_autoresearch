# Analysis: Erdős Problem 123 — Antichain Sum Reformulation

## Problem Statement
The theorem claims: for any three pairwise coprime integers a, b, c > 1, the set
{a^i · b^j · c^k : i, j, k ∈ ℕ} is **d-complete** — every sufficiently large integer
can be written as a sum of distinct elements from this set that form a **divisibility antichain**
(no element divides another).

## Reformulation Assessment: Easier or Harder?

**This reformulation is approximately equivalent in difficulty to the original Erdős Problem 123.**

The "antichain sum representation drawn from the triple-power semigroup" is essentially
a restatement of d-completeness for multiplicative semigroups generated by three coprime
bases. The reformulation makes the structure more explicit (divisibility antichains in ℕ³
correspond to incomparable exponent vectors), but does **not** simplify the core mathematical
difficulty, which lies in:

1. **Asymptotic completeness**: Showing that the smooth numbers (products of powers of a, b, c)
   are dense enough for subset sum representation of all large integers.
2. **Antichain conversion**: Converting arbitrary subset sum representations into ones satisfying
   the divisibility antichain constraint.

The reformulation is **slightly easier to work with formally** because:
- The exponent-vector view via `powTriple_dvd_iff` cleanly characterizes divisibility
- The `level_antichain` construction provides automatic antichains from any bc-level
- The 3D structure makes incomparability conditions more transparent

But it is **no easier mathematically** because the fundamental challenge — showing that
the combinatorial structure supports representation of ALL large integers — remains intact.

## Key Discovery: Two Generators Are Insufficient

During the proof attempt, we discovered computationally that the **two-generator** version
{b^j · c^k : j, k ≥ 0} is **NOT d-complete** in general. For b=3, c=5, many integers
in [400, 500] have no antichain sum representation from {3^j · 5^k}. This means:

- All three generators are essential for d-completeness
- The proof CANNOT reduce to the two-generator case
- The interplay between all three coprime generators is crucial

## What Was Proved

### Fully proved lemmas (14 total):
1. `one_mem_powTripleSet` — 1 ∈ PowTripleSet
2. `mul_a_mem`, `mul_b_mem`, `mul_c_mem` — closure under generator multiplication
3. `pow_mem_powTripleSet_a/b/c` — generator powers are in the set
4. `powTriple_dvd_iff` — **Key structural lemma**: divisibility ↔ componentwise ≤ on exponents
5. `pos_of_mem_powTripleSet` — all elements are positive
6. `singleton_antichain` — singletons are trivial antichains
7. `generators_antichain` — {a, b, c} is a divisibility antichain
8. `singleton_sum` — sum of singleton finset
9. `mem_has_antichain_rep` — every smooth number has a trivial antichain representation
10. `powTripleSet_infinite` — the set is infinite
11. `frobenius_two` — Frobenius/Chicken McNugget theorem for two coprime generators
12. `level_antichain` — **Key**: elements at the same bc-level with varying a-exponents always form antichains
13. `level_mem_powTripleSet` — level elements are in PowTripleSet
14. `IsDComplete.mono` — d-completeness is monotone under set inclusion
15. `ab_ac_incomparable` — elements a^i·b and a^j·c are always divisibility-incomparable
16. `pair_ab_ac_antichain` — pairs {a^i·b, a^j·c} are antichains

### Main theorem structure:
The main theorem `erdos_123_d_complete_sequences` is **proved modulo two key lemmas**:
- `smooth_subset_sum_complete`: every large integer is a sum of distinct smooth numbers
- `subset_sum_to_antichain`: any such representation can be converted to an antichain

### Remaining sorries (2):
Both capture genuine mathematical depth:

1. **`smooth_subset_sum_complete`**: Requires showing {a,b,c}-smooth numbers are
   "asymptotically complete" — every large integer is a sum of distinct smooth numbers.
   This needs analytic number theory estimates on smooth number density and gap distribution.
   For a=2 it follows from binary representation; for a≥3 it requires the full three-generator
   density argument.

2. **`subset_sum_to_antichain`**: Requires showing that the 3D exponent structure of
   PowTripleSet provides enough flexibility for antichain rearrangement. This is where
   the pairwise coprimality of all three generators is essential.

## Computational Verification

For a=2, b=3, c=5: every integer from 1 to 200 has an antichain representation (verified).
For a=3, b=5, c=7: all integers ≥ 93 have representations; the non-representable integers
below 93 are {2, 4, 6, 11, 13, 17, 18, 19, 20, 23, 29, 33, 37, 43, 51, 92}.

## Proof Difficulty Classification

This problem sits at the boundary of what's formalizable without deep analytic number theory
infrastructure in Mathlib. The main obstacles are:
- No formalized theory of smooth number density in Mathlib
- No formalized "complete sequence" theory for asymptotic subset sum coverage
- The antichain constraint adds a combinatorial layer on top of additive number theory

**Estimated difficulty**: Research-level formalization, comparable to a published paper.
