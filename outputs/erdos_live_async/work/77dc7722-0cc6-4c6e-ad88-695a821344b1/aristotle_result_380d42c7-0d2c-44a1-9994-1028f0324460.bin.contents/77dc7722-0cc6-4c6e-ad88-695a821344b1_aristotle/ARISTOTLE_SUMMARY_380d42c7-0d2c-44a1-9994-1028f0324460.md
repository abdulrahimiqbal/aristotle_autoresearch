# Summary of changes
## Summary

### What was done

The file `AristotleWorkspace.lean` formalizes infrastructure around **Erdős Problem 123** on d-complete sequences, reformulating the conjecture at the extremal boundary case (2, 3, 5).

### Proved results (sorry-free)

All infrastructure and partial results are fully proved (18 lemmas/theorems):

**§2. Membership lemmas**: `one_mem_powTripleSet`, `pow_a_mem`, `pow_b_mem`, `pow_c_mem`, `triple_mem`

**§3. Divisibility-extraction lemmas** (reusable across all triples):
- `coprime_pow_not_dvd_pow` — Positive power of one coprime base never divides a positive power of the other
- `coprime_pow_pow` — Powers of coprime bases remain coprime
- `same_base_dvd` — Distinct powers of the same base always have divisibility
- `mixed_coprime_not_dvd` — Elements with incomparable exponent vectors form antichains (**strongest reusable lemma**)

**§4. Antichain constructors**: `singleton_antichain`, `pair_coprime_antichain`, `triple_coprime_antichain`, `empty_antichain`, `antichain_subset`

**§5. Covering**: `pow2_sum_representation` (binary representation baseline)

**§6. Partial d-completeness**:
- `coprime_sum_d_repr` — Numbers of the form a^i + b^j (coprime bases) have antichain representations
- `infinitely_many_d_representable` — Infinitely many naturals are d-representable from any PowTripleSet

**§7. Computational verification**: `boundary_verified_2000` — Every n from 1 to 2000 has an antichain representation from 5-smooth numbers (verified by `native_decide`)

### Remaining sorries (2)

Both are **open conjectures** (Erdős Problem 123, a well-known open problem in combinatorial number theory):
- `erdos_123_boundary_2_3_5` — d-completeness for the extremal triple (2, 3, 5)
- `erdos_123_d_complete_sequences` — d-completeness for all coprime triples

### Discovery: Reusable lemmas strong enough to promote

The module docstring contains a detailed analysis. The key findings:

1. **`mixed_coprime_not_dvd`** is the strongest candidate for promotion — it enables construction of arbitrarily large antichains from any coprime pair and its proof is fully parameter-independent.

2. **`coprime_pow_not_dvd_pow`** is the fundamental antichain ingredient that applies identically across all coprime triples.

3. **`pair_coprime_antichain` / `triple_coprime_antichain`** are the most frequently used workhorse constructors across all concrete triple calculations.

4. **`antichain_subset`** is essential structural glue for extracting sub-antichains.

5. **`pow2_sum_representation`** provides the baseline covering argument before antichain refinement.

These lemmas recur across all parameter triples because they depend only on coprimality and the multiplicative structure of PowTripleSet, not on specific values of (a, b, c).
